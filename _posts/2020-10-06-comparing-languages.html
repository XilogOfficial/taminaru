Comparing programming languages with pure math

how close are they really?

Programming languages are usually described as being close to math and mathematical notation. However, to what extent does that apply?

For this comparison we will use the C programming language.

NOTE: Instead of syntax, we will compare how the language works.

Numbers

In math we have the following number groups:

  • Natural numbers (N)
  • Integers (Z)
  • Rational numbers (Q)
  • Real numbers (R)

*excluding whole numbers, since they are the same as natural numbers

In programming languages we usually have the following number groups:

  • Integers
  • Floats

Integers correspond exactly with math.
Floats comprise of all rational and irrational numbers, plus integers.

Let's see an example in C:

/* Equivalents to integers */
int a   = 4; /* usually 4 bytes */
short b = 2; /* usually 2 bytes */
long  c     = 4; /* usually 4 bytes */
long long d = 8; /* usually 8 bytes */

/* Floats (and doubles) */
float x = 3.2f /* NOTE: the 'f' denotates a floating point literal */
double y = 0.0;

C is, in general, not very off. The major differences are the need of having multiple data types connected with the computer's hardware.

Functions

In mathematical notation, one may describe a function like so:

f(x) = x + 1

This function takes a variable, x, and adds 1 to it.
Let's compare:

int f(x) {
  putchar((x + 1));
}

C may not be close to mathematical notation, but the concept is still very similar. We are making a function, which takes the variable x and adds 1 to it.

Sets

The equivalent of sets in C are arrays.

/* This array holds double numbers */
double a[] = { 100.0, 42.0, 1.0, 0.0 };

/* This array holds 3 integers */
int b[3] = { 1, 2, 3 };

Sequences

In mathematical notation, sequences are usually written in this form:

(0, 1, 2, 3, 4) — finite sequence
(0, 1, 2, ...)  — infinite sequence

What differentiates sequences from arrays is that sequences must be in order.

C does not have sequences (not to be confused with escape sequences; however, we can get really close by using arrays.

int i;

/* Sequence from 0 to 100 (including 0 and 100) */
for (i = 0; i <= 100; ++i) {
    printf("%d ", i); /* Prints the sequence */
}
